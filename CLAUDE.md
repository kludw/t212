# CLAUDE.md

Build/test/regen reference for contributors (and Claude Code).

## Layout

- `client.go` — `Client`, `ClientOpts`, `NewClient`, `Close`, `Snapshot`, `do`, all endpoint methods, the auto-started position watcher.
- `pagination.go` — `*Iter` helpers (Go 1.23 range-over-func) for the cursor-paginated history endpoints.
- `reports.go` — `WaitForReport` poller for CSV exports.
- `accessors.go` — hand-written nil-safe `Get*` methods over the generated types. Lives in its own file so it survives `models.gen.go` regeneration.
- `constructors.go` — `NewMarketRequest` / `NewLimitRequest` / etc. Same regen-survival reasoning.
- `errors.go`, `defs.go` — error sentinels and package constants.
- `models.gen.go` — generated from the upstream OpenAPI spec; do not hand-edit. Regenerated by `scripts/generate-models.sh`.
- `example/` — runnable usage examples; one subdir per scenario.
- `scripts/test.sh`, `scripts/generate-models.sh` — local entry points.
- `.github/workflows/` — CI: `test.yml`, `lint.yml`, `coverage.yml`, `regen-models.yml`.

## Common tasks

```sh
# Build everything
go build ./...

# Run tests (race + 25 reps + failfast, same as CI)
bash scripts/test.sh

# Quick test loop
go test ./...

# Coverage on hand-written code only
go test -coverprofile=cov.out ./...
grep -v "models\.gen\.go" cov.out | go tool cover -func=/dev/stdin | tail -1

# Lint
golangci-lint run

# Regenerate models from the live T212 spec
bash scripts/generate-models.sh
```

## Notes for changes

- The position watcher auto-starts in `NewClient`. Tests that exercise the watcher must construct via `NewClient` (using `ClientOpts.BaseURL` to point at an `httptest.Server`); the `newWatcherClient` helper in `client_test.go` does the wiring. Tests for individual endpoint methods bypass `NewClient` via `newTestClient` so they don't poll.
- `nextPagePath` returned by the history endpoints is a path under `/api/v0/...`; the iterators parse the `cursor` query param out of it and reuse the existing typed methods. See `nextPageCursor` in `pagination.go`.
- `accessors.go` and `constructors.go` are hand-written precisely because `models.gen.go` is regenerated weekly by CI. If you add new accessor/constructor functions, put them in those files (or new files alongside) — never in the generated file.
- New endpoint methods belong in `client.go` and follow the existing shape: take `ctx`, build query params via `url.Values`, delegate to `c.do`. Document the rate limit in the godoc.
- New error sentinels go in `errors.go`. Map HTTP statuses in `statusError`.
- Watcher behaviour is documented in the `ClientOpts` godoc. Errors from the polling loop are passed to `OnPollError` (if set) and otherwise dropped; the watcher does not stop on errors.
- The Trading 212 API is **not idempotent** for order placement — never auto-retry POST `/orders/*` calls.
